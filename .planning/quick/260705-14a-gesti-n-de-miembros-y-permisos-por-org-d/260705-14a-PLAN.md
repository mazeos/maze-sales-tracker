El comportamiento externo de POST /api/orgs NO cambia (mismos status, mismos mensajes, `admin_password` solo si `generated && createdAuth`).

2. **`GET /api/orgs/{orgId}/members`** → función `listOrgMembers(req, res, sa, orgId)`: fetch `st_orgs?id=eq.{orgId}&select=id` → si no existe, 404 'Esa organización no existe'. Fetch `st_profiles?org_id=eq.{orgId}&select=id,name,role,active,ghl_user_id&order=name.asc`. Para cada perfil, email vía `getAuthEmail(p.id, emailCache)` con UN Map por request. Respuesta 200: `{members:[{id, name, email, role, active: p.active !== false, ghl: !!p.ghl_user_id}]}`.

3. **`PATCH /api/orgs/{orgId}/members/{uid}`** → función `patchOrgMember(req, res, sa, orgId, uid)`: body JSON `{role?, active?}`; al menos uno presente sino 400. `role` debe ser uno de `['admin','setter','triage','closer']` sino 400. `active` debe ser boolean sino 400. Verificar que el perfil existe y `org_id === orgId` sino 404 'Ese miembro no pertenece a esa organización'. **Protección último admin:** leer los perfiles de la org (`select=id,role,active`); simular el cambio sobre el target; si después del cambio la org queda con CERO perfiles `role==='admin' && active !== false` → 400 'La organización no puede quedar sin admin' (cubre tanto degradar rol de un admin como desactivarlo). Aplicar PATCH a `st_profiles?id=eq.{uid}` con los campos presentes. Luego, si `active === false` → ban best-effort (patrón deleteMember, `ban_duration:'87600h'`, try/catch vacío); si `active === true` → unban best-effort (`ban_duration:'none'`). Respuesta 200 `{ok:true}`. Log estilo `[api] PATCH /api/orgs/.../members super=${sa.email} ... -> status`.

4. **`POST /api/orgs/{orgId}/admins`** → función `addOrgAdmin(req, res, sa, orgId)`: body `{name, email, password?}` con las MISMAS validaciones que createOrg (nombre requerido, email regex `/^\S+@\S+\.\S+$/`, password opcional min 8; si vacía autogenerar 14 chars con el mismo patrón, flag `generated`). Verificar que la org existe (404 si no). Llamar `provisionOrgAdmin({orgId, name, email, password})`; si `!ok` propagar status+error. Respuesta 200: `{admin_email: email, existing_account: existingAccount}` + `admin_password` SOLO si `generated && createdAuth`.

5. **Router**: agregar los matches justo después del bloque `/api/orgs` existente (línea ~2095), con el patrón regex del archivo:
